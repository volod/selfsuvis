## Vision

### 10x Check
The 10x version makes the self-improvement loop **fully automatic**: CVAT webhook fires → DB marks frames annotated → CVAT webhook handler itself checks annotated frame count and triggers fine-tuning when the threshold is crossed (event-driven, not poll-driven — no dependency on a new video being indexed) → fine-tuning kicks off in a durable background job (PostgreSQL job queue, not a FastAPI background task) → checkpoint is evaluated against a held-out 10% validation set using top-1 accuracy → if accuracy improves over the previous checkpoint, the new checkpoint is promoted to `DINO_CHECKPOINT` → DINOEmbedder is reloaded safely (write lock around all inference calls) → a durable re-embedding sweep job re-embeds all historical frames → mission summary reports show "model v3 improved top-1 accuracy by 12% on your annotated set." Zero human intervention after the initial annotation batch.

### Platonic Ideal
An outdoor robotics team deploys selfsuvis on day 1. They fly 10 missions before doing any annotation. The active learning pipeline has already tagged the 50 highest-uncertainty frames from each mission. On day 5, an engineer spends 2 hours in CVAT annotating those 500 frames. CVAT webhook fires. By day 5 evening, the DINOv3 backbone has been fine-tuned on their domain — the improvement is live in the search index with zero ops work. They fly 10 more missions. The next annotation batch is even better-targeted. The moat compounds.

## Accepted Scope — Implementation Contracts

### 1. DB-native frame discovery

**Interface: `AnnotatedFrameDataset.from_db(conn, frames_base_dir, label_names=None)`**

```python
@classmethod
async def from_db(
    cls,
    conn,                       # asyncpg connection
    frames_base_dir: str,       # settings.FRAMES_DIR (top-level, recursive scan covers all missions)
    label_names: Optional[List[str]],  # If None, derived internally via SELECT DISTINCT cvat_label
    transform: transforms.Compose,
    two_views: bool = True,
) -> Optional["AnnotatedFrameDataset"]:
    # 1. If cvat_label column missing: log ERROR, return None (caller skips training)
    #    Check: SELECT column_name FROM information_schema.columns
    #           WHERE table_name='frames' AND column_name='cvat_label'
    # 2. If label_names is None: run SELECT DISTINCT cvat_label FROM frames
    #    WHERE al_tag='annotated' AND cvat_label IS NOT NULL ORDER BY cvat_label
    # 3. Query: SELECT frame_path, cvat_label FROM frames
    #    WHERE al_tag='annotated' AND cvat_label IS NOT NULL
    # 4. Build frame_labels dict keyed by frame_id (NOT basename): join frames.frame_id
   #    against cvat_tasks.frame_id to get per-frame labels — no basename matching
```

**Schema gap:** The `frames` table has `al_tag` but no `cvat_label` column. The CVAT webhook sets `al_tag='annotated'` but discards the label strings. To make `from_db` work, the webhook handler must fetch labels from CVAT after the annotation event fires.

**CVAT label fetch-back spec:** CVAT webhook payloads contain task/job identifiers but NOT per-frame label assignments inline. The webhook handler must call back to the CVAT REST API to retrieve them:

1. Parse `task_id` from webhook JSON payload
2. Call `GET {CVAT_URL}/api/tasks/{task_id}/annotations?format=CVAT+1.1` with `Authorization: Token {CVAT_API_TOKEN}` header
3. Parse the returned CVAT XML 1.1 (same format as `CvatAnnotationParser` already handles) — reuse `CvatAnnotationParser`
4. For each `<image name="...">` in the XML, match `basename(name)` against `frames.frame_path`; store majority-vote label as `cvat_label`

**New config vars:** `CVAT_API_TOKEN = _env("CVAT_API_TOKEN", "")` — CVAT REST API token for label fetch-back. If empty, `_mark_frames_annotated` logs WARNING and skips label storage (frames marked `al_tag='annotated'` but `cvat_label=NULL` → `from_db` skips them).

**Option A (preferred):** Add `cvat_label TEXT` column to `frames`. CVAT webhook handler fetches labels from CVAT API and stores them. `from_db` queries `WHERE al_tag='annotated' AND cvat_label IS NOT NULL`.
**Option B (fallback):** Keep a separate `cvat_annotations` table mapping `(frame_id, label)`. `from_db` JOINs against it.

**`from_db` fallback when column is missing:** `from_db` checks for `cvat_label` column existence at call time. If absent, it logs `ERROR: cvat_label column missing — fine-tuning skipped; run migrate_postgres.py` and returns `None`. Callers (`_maybe_trigger_finetune`) must handle `None` return and short-circuit without training.

**`label_names` derivation:** `from_db` runs `SELECT DISTINCT cvat_label ... ORDER BY cvat_label` internally when `label_names=None` is passed. Callers never need to pre-compute the label vocabulary — the method is self-contained.

**CLI change:** `--from-db` flag on `supervised_finetune_dino.py`. When set, `--cvat-xml` becomes optional (ignored); `--frames-dir` must be `settings.FRAMES_DIR`. `DATABASE_URL` must be set; if not, exit with clear error message.

### 2. Auto-trigger from CVAT webhook handler

**Trigger point:** `app/routers/cvat.py` `_mark_frames_annotated()` — AFTER updating `al_tag`, check:

```python
# Check whether to auto-trigger fine-tuning
if settings.SUP_AUTO_TRIGGER:
    total_annotated = await conn.fetchval(
        "SELECT COUNT(*) FROM frames WHERE al_tag = 'annotated'"
    )
    if total_annotated >= settings.MIN_ANNOTATED_FRAMES:
        await _maybe_trigger_finetune(conn, total_annotated)
```

**`_maybe_trigger_finetune` contract:** This function MUST NOT run training inline — that would block the FastAPI event loop for the entire duration of multi-epoch GPU training. Instead it enqueues a durable PostgreSQL job and returns immediately. It also checks a retrain watermark to avoid re-triggering after minimal incremental annotations:

```python
async def _maybe_trigger_finetune(conn, total_annotated: int) -> None:
    async with _finetune_lock:
        # Check watermark: only retrain if enough new annotations since last run
        watermark = await conn.fetchval(
            "SELECT value FROM system_state WHERE key='last_retrain_watermark'"
        ) or 0
        if (total_annotated - int(watermark)) < settings.MIN_NEW_ANNOTATED_SINCE_RETRAIN:
            logger.debug("retrain watermark not reached (%d since last retrain)", total_annotated - int(watermark))
            return
        # Check if a finetune job is already running or queued
        existing = await conn.fetchval(
            "SELECT id FROM jobs WHERE type='supervised_finetune' AND status IN ('queued','running') LIMIT 1"
        )
        if existing:
            logger.info("fine-tuning job already queued/running, skipping trigger")
            return
        job_id = await enqueue_job(conn, type="supervised_finetune", payload={
            "total_annotated": total_annotated,
            "checkpoint": settings.DINO_CHECKPOINT,
        })
        logger.info("auto-triggered fine-tuning job %s (%d annotated frames)", job_id, total_annotated)
# Worker updates watermark after successful finetune job:
# UPDATE system_state SET value=$1 WHERE key='last_retrain_watermark'
```

**`system_state` table:** `CREATE TABLE IF NOT EXISTS system_state (key TEXT PRIMARY KEY, value TEXT)` — add to `migrate_postgres.py`.
**New config var:** `MIN_NEW_ANNOTATED_SINCE_RETRAIN = _env_int("MIN_NEW_ANNOTATED_SINCE_RETRAIN", 100)` — minimum new annotations since last retrain before triggering again.

The worker's main dispatch loop handles `type='supervised_finetune'` — add `async def handle_supervised_finetune_job(conn, job)` in `worker/main.py` (or `pipeline/supervised_finetune_worker.py`). That handler calls `from_db(conn, ...)` then `run_supervised_finetune(cfg)` in a separate thread (`asyncio.run_in_executor`) to avoid blocking the event loop even inside the worker.

**Concurrency guard:** Module-level `asyncio.Lock` in `cvat.py` — `_finetune_lock`. Gates job enqueue, not training execution. Prevents two rapid webhook firings from enqueueing duplicate jobs.

**Config additions:**
- `SUP_AUTO_TRIGGER = false` (opt-in; must be explicitly enabled)
- `MIN_ANNOTATED_FRAMES = 500` (minimum before triggering)

**Documentation:** Add new env vars (`SUP_AUTO_TRIGGER`, `MIN_ANNOTATED_FRAMES`) to CLAUDE.md under "Supervised fine-tuning" — this is a documentation-only task, not an implementation gate.

### 4. Hot-reload endpoint

**App state change in `app/state.py`:** Add `dino_model_lock = asyncio.Lock()` — a mutual-exclusion lock used only to serialise concurrent reload requests. Inference calls do NOT acquire this lock; they capture a reference to `app.state.dino_model` at the start of each request. The reload replaces `app.state.dino_model.backbone` in-place while any in-flight inference completes on the old reference.

**Router placement:** Add endpoint to the existing `app/routers/admin.py` router (`prefix='/admin'`). Requires `X-API-Key` header (inherits from admin router's auth dependency).

**Endpoint:**

```
POST /admin/reload-model
Body: {"checkpoint": "/path/to/dino_sup_best.pt"}
```

- Validates file exists and is readable
- Acquires `dino_model_lock` (mutual-exclusion; serialises concurrent reloads only)
- Loads new checkpoint into existing `app.state.dino_model.backbone` in-place
- Writes new checkpoint path to `settings.SUP_CHECKPOINT_DIR/active_checkpoint.txt` so the next API restart auto-loads it without requiring `DINO_CHECKPOINT` env var change
- Releases lock
- Returns `{"status": "ok", "checkpoint": "...", "reloaded_at": ...}`
- On error: returns 500 with detail; old model remains active; `active_checkpoint.txt` NOT written

**Worker triggers reload via HTTP.** The fine-tune worker calls `POST /admin/reload-model` with the checkpoint path. The worker and API are separate OS processes — direct module sharing is not possible. Worker authenticates using `settings.API_KEY` in the `X-API-Key` header.

**`DINOEmbedder` startup:** At load time, if `DINO_CHECKPOINT` env var is empty, check `settings.SUP_CHECKPOINT_DIR/active_checkpoint.txt`. If the file exists and points to a readable file, load that checkpoint; if the file is missing or points to a nonexistent path, log WARNING and use pretrained hub weights. This ensures hot-reloaded checkpoints survive API restarts without env var changes.

**New config var:** `SUP_CHECKPOINT_DIR = _env("SUP_CHECKPOINT_DIR", os.path.join(DATA_DIR, "checkpoints", "supervised"))` — already listed in `pipeline/config.py`; confirm it is added there.

### 5. Re-embedding sweep (durable job)

**Durability:** Enqueue via the existing PostgreSQL job queue (`pipeline/job_db.py`) rather than `BackgroundTasks`. The sweep is idempotent: tracks the last re-embedded `qdrant_id` in the job's `progress_json`. On restart, resumes from last checkpointed point.

**Endpoint:**

```
POST /admin/reembed-all
```

- If a re-embedding job is already in progress (job with top-level `type='reembed'` and `status IN ('queued','running')`): return 409 Conflict
- Otherwise: enqueue via `enqueue_job(conn, type='reembed', payload={"checkpoint": settings.DINO_CHECKPOINT})` — `type` is the top-level job column, consistent with `supervised_finetune` job
- Returns job_id immediately; use `GET /jobs/{id}` for progress

**Worker handles job type `reembed`:** Add `async def handle_reembed_job(conn, job)` in `worker/main.py` (or `pipeline/reembed.py`), dispatched from the worker's main job loop on `job['type'] == 'reembed'` (top-level `type` column, not `payload['type']`). Scrolls Qdrant in batches of 100; loads frame JPEG; re-embeds with current DINOEmbedder; upserts `dino` named vector in-place. Progress: `{"processed": N, "total": M, "last_id": ...}`. On exception per-point: if `frame_path` is missing from payload, log `WARNING: point {id} has no frame_path — skipping` and continue to next point (do NOT abort the entire job). On unhandled exception: marks job `error`, logs full context.

**`run_supervised_finetune` threading note:** When called from the worker via `asyncio.run_in_executor`, set `DataLoader(num_workers=0)` to avoid subprocess-from-thread deadlock (known issue on Python 3.12+ with forked processes). If DataLoader worker parallelism is needed, use a dedicated `ProcessPoolExecutor` with a single worker instead.

**Scaling note for eval_accuracy:** Gallery rebuild is O(N) backbone forward passes per epoch; acceptable for ~500 frames. Revisit for N > 5,000 (use FAISS flat index or subsample gallery to 1,000 random train samples).

**frames_dir note:** Re-embedding uses `frame_path` from Qdrant payload (already stored per-frame); no `frames_dir` dependency. Legacy points that predate `frame_path` in the payload schema are skipped gracefully (logged as warnings).

## Architecture Diagram

```
CVAT ANNOTATION COMPLETE
      │
      ▼
POST /webhook/cvat  [FastAPI async handler — must return quickly]
      │
      ├─► _mark_frames_annotated() → UPDATE frames SET al_tag='annotated', cvat_label=...
      │
      └─► if SUP_AUTO_TRIGGER=true:
               COUNT(*) WHERE al_tag='annotated' ≥ MIN_ANNOTATED_FRAMES?
                    │ yes
                    ├─► _finetune_lock acquired? → skip (job already queued/running)
                    │
                    └─► enqueue job type='supervised_finetune' → returns immediately
                              │
                              ▼ (worker process, separate from API)
                         handle_supervised_finetune_job()
                              │
                              ▼
                         from_db(conn, FRAMES_DIR) → AnnotatedFrameDataset
                         [fallback: None if cvat_label col missing → job logs ERROR, exits]
                              │
                              ▼
                         size gate: total < SUP_MIN_EVAL_GATE_FRAMES?
                         → skip eval gate, promote unconditionally
                              │ else
                              ▼
                         stratified 90/10 split
                         SupConLoss training loop (run_in_executor — non-blocking)
                         eval_accuracy() on hold-out each epoch
                         [backbone CLS embeddings, L2-norm, 1-NN classifier]
                              │ if accuracy improved (or gate skipped)
                              ▼
                         promote → dino_sup_best.pt
                         write active_checkpoint.txt
                              │
                              ▼
                         POST /admin/reload-model (asyncio.Lock on dino_model)
                              │
                              ▼
                         enqueue job type='reembed'
                              │
                              ▼ (worker process — same or different job)
                         handle_reembed_job(): scroll Qdrant, re-embed, upsert
                         (durable: progress_json checkpoints last_id, restartable)
                         [missing frame_path → skip point with WARNING, continue]
```

## Reviewer Concerns (iteration 3 findings — max iterations reached; persisted as-is)

These issues were surfaced in the 3rd adversarial review pass. Low-severity items acceptable to resolve during implementation:

- **asyncio.Lock multi-replica caveat (low):** The asyncio.Lock in cvat.py prevents in-process duplicate enqueues only. In a multi-replica API deployment, the SQL query (`WHERE type='supervised_finetune' AND status IN ('queued','running')`) is the authoritative dedup guard. Implementers should not rely on the asyncio.Lock for multi-replica safety.
- **FRAMES_DIR config var (low):** `settings.FRAMES_DIR` is used in `from_db` and the CLI. Confirm this field exists in `pipeline/config.py` (it does — `FRAMES_DIR = _env("FRAMES_DIR", os.path.join(DATA_DIR, "frames"))`).
- **Manual reembed endpoint scope (low):** `POST /admin/reembed-all` is an ops convenience endpoint not required for the automated loop. Could be deferred to v2; kept in scope as it adds minimal complexity.
