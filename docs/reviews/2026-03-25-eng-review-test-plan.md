## Key interactions to verify

- CVAT webhook → `_mark_frames_annotated` → `_maybe_trigger_finetune` → enqueue `type='supervised_finetune'` job
- Worker picks up `type='supervised_finetune'` job → calls `run_supervised_finetune` in executor → eval gate passes → calls `POST /admin/reload-model`
- Worker picks up `type='reembed'` job → processes frames in batches of 256 → checkpoints `progress_json.last_offset` after each batch
- `POST /admin/reload-model` acquires lock → loads checkpoint → swaps `app.state.dino_model` reference → writes `active_checkpoint.txt`
- `POST /admin/reembed-all` enqueues job, returns `job_id`

## Edge cases

### AnnotatedFrameDataset.from_db
- `mission_id=None` returns all annotated frames across all missions
- Empty result (no annotated frames in DB) → dataset length 0, DataLoader must not crash
- DB connection error → propagates exception, not silently swallowed

### _maybe_trigger_finetune
- Annotated count below `MIN_ANNOTATED_FRAMES` → no job enqueued
- `SUP_AUTO_TRIGGER=false` → no job enqueued even above threshold
- Retrain watermark: `total_annotated - last_retrain_watermark < MIN_NEW_ANNOTATED_SINCE_RETRAIN` → no job enqueued
- Job already `queued` or `running` → second job NOT enqueued (dedup guard)
- DB error in `create_job` → logged, webhook still returns 200

### Checkpoint gate in run_supervised_finetune
- `best_accuracy >= SUP_EVAL_GATE_THRESHOLD` → `accepted=True`, checkpoint saved
- `best_accuracy < SUP_EVAL_GATE_THRESHOLD` → `accepted=False`, checkpoint NOT promoted
- Dataset too small (`< SUP_MIN_EVAL_GATE_FRAMES`) → reject with error, return `accepted=False`, no training
- Single-class dataset (stratified split impossible) → graceful fallback or clear error
- Returns dict `{"path", "best_accuracy", "epochs", "accepted"}` not bare string

### POST /admin/reload-model
- Checkpoint path not found on disk → 400
- Lock already held (concurrent reload) → 409
- Weights file corrupt / load raises → 500, existing `dino_model` reference unchanged
- No `checkpoint` in request body → fallback to `active_checkpoint.txt`; if file missing → 400

### POST /admin/reembed-all
- Happy path → returns `{"job_id": "<uuid>"}`, `type='reembed'` job in DB
- Behaviour when reembed already in progress: to be defined by implementation (reject or queue)

### handle_reembed_job (worker)
- `last_offset` in `progress_json` → resume from that frame index, skip earlier frames
- Qdrant upsert failure mid-batch → job marked `error`, `last_offset` preserved for resume
- Zero frames in DB → job finishes with `frames_reembedded=0`, status `finished`
- Batch size 256: assert only one Qdrant upsert call per batch, not per frame

### Worker type dispatch
- Unknown `type` value → job marked `error`, warning logged
- `type=None` (legacy index jobs without type column) → routed to existing index handler

### job_db_pg type column (REGRESSION)
- `create_job(type='finetune')` → `fetch_job` returns `type='finetune'`
- `MockConn` in `test_job_db_postgres.py` must include `type` field or `_row_to_dict` silently drops it

## Critical paths

1. **Happy loop**: annotate 500 frames in CVAT → webhook fires → retrain triggers → checkpoint promoted → model reloads → search uses new embeddings (requires: cvat_label migration, retrain watermark, all 5 scope items wired together)
2. **Checkpoint gate rejection**: low-quality run must NOT promote — existing model must be unchanged after gate fail
3. **Reembed resume**: kill worker mid-sweep at offset 5000 → restart → verify frames 0-4999 not re-embedded (progress_json checkpoint)
4. **Concurrent reload protection**: two simultaneous `POST /admin/reload-model` requests → second must get 409, not corrupt model state
