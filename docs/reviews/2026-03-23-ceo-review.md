# CEO Plan: Outdoor Autonomy Perception Stack — Living World Model
Generated by /plan-ceo-review on 2026-03-23
Branch: main | Mode: SCOPE_EXPANSION
Repo: selfsuvis

## Vision

### 10x Check
Not a queryable video archive — a **living, persistent outdoor world model**. Each new mission auto-registers against the existing global 3D map. The system detects environmental change over time ("the northern path had dense vegetation in mission 3, now it's clear in mission 7"). The robot queries the world model **advisorily** during flight: "what did we previously see near my current GPS position?" (200ms REST; advisory use only — hard real-time obstacle avoidance requires onboard compute, out of scope). Active learning closes the loop: per-frame uncertainty scores (v1) feed CVAT annotation queues (v2), which feed fine-tuning (v2). The v1 architecture writes data in the exact format v2 training requires — no retrofit.

> **Active learning in v1:** `pipeline/active_learning.py` computes `active_learning_score = 0.6 × DINOv3_embedding_distance_from_nearest_centroid + 0.4 × (1 − caption_confidence)`. Cluster centroids: k-means (k=20) run over all mission DINOv3 embeddings after each mission, updated incrementally; centroids stored in PostgreSQL `embedding_clusters` table (`cluster_id INT, centroid_json JSONB`). Caption confidence from Florence-2 model output logits (`pipeline/florence_model.py`). `al_tag` values in v1: `needs_annotation` (top-50 by score, `AL_TAG_K` env var), `novel` (embedding distance > 0.5 from any centroid), `none` (all others). `al_tag` values `uncertain`, `annotated`, `used_for_training` reserved for v2 migration — stored as CHECK constraint but never assigned in v1.

## Architecture Context

**PostgreSQL replaces SQLite entirely** (ADR-0001). No hybrid. Docker Compose adds `postgres:16`. `pipeline/job_db.py` and `pipeline/processed_db.py` rewritten to `asyncpg`/`psycopg2`.

**Migration:** `jobs.db` and `processed.db` data is **not migrated** — both databases are ephemeral job queues and dedup registries; existing jobs/state wiped during cutover. Migration procedure: (1) stop worker, (2) `docker compose up postgres`, (3) run `scripts/migrate_postgres.py` (creates tables), (4) restart API + worker. Zero data loss risk: processed_files records only mean files would be re-indexed; run `scripts/reset_qdrant.sh` anyway since DINOv3 vector switch invalidates existing Qdrant data.

**New system dependencies:**
- `pycolmap` — requires COLMAP C++ shared libs; custom Docker base image or multi-stage build needed (add to `Dockerfile.worker`; estimated +800MB to image)
- `nerfstudio splatfacto` — requires CUDA + tinycudann build; separate `Dockerfile.nerfstudio` based on `dromni/nerfstudio`; exposed as `docker-compose.override.yml` optional service (GPU machines only)
- `Florence-2` (`microsoft/Florence-2-large`) — `transformers >= 4.41`; model checkpoint ~900MB; add to GPU worker
- `Open3D` — for Phase 2 ICP (v2 only; do not add to v1 Docker image)

**Core PostgreSQL schemas:**

```sql
-- Migrated from SQLite:
CREATE TABLE jobs (...);            -- same columns as existing jobs.db schema
CREATE TABLE processed_files (sha256 TEXT PRIMARY KEY, video_id TEXT, indexed_at TIMESTAMPTZ);

-- New tables:
CREATE TABLE embedding_clusters (
  cluster_id SERIAL PRIMARY KEY,
  centroid_json JSONB NOT NULL,     -- float array of DINOv3 embedding dimension
  updated_at TIMESTAMPTZ
);

CREATE TABLE missions (
  mission_id TEXT PRIMARY KEY,
  video_id TEXT NOT NULL,           -- opaque identifier; no FK (no videos table in v1)
  source_path TEXT,
  ingest_mode TEXT,
  started_at TIMESTAMPTZ,
  completed_at TIMESTAMPTZ,
  frame_count INT,
  pose_status TEXT CHECK (pose_status IN ('pending','success','failed')),
  map_status TEXT CHECK (map_status IN ('pending','success','failed','skipped')),
  report_path TEXT                  -- nullable; set on report generation
);

CREATE TABLE frames (
  frame_id TEXT PRIMARY KEY,
  video_id TEXT NOT NULL,
  mission_id TEXT REFERENCES missions(mission_id),
  timestamp_ms BIGINT,
  filepath TEXT,
  caption TEXT,                     -- Florence-2 output; nullable until captioning runs
  caption_confidence FLOAT4,
  active_learning_score FLOAT4,
  al_tag TEXT CHECK (al_tag IN ('needs_annotation','uncertain','novel','annotated','used_for_training','none')),
  -- uncertain/annotated/used_for_training: reserved for v2; never assigned in v1
  pose_json JSONB,                  -- pycolmap {rotation_matrix, translation, intrinsics} or null
  gps_json JSONB,                   -- {lat, lon, alt} from video container metadata or null
  global_pose_json JSONB,           -- {tx,ty,tz,qx,qy,qz,qw} ENU frame; null until global map registration
  created_at TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_frames_mission ON frames(mission_id);
CREATE INDEX idx_frames_al_tag ON frames(al_tag);
CREATE INDEX idx_frames_al_score ON frames(active_learning_score DESC);
```

**3DGS reconstruction pipeline** (ADR-0004, in scope for v1): Frame extraction → `pipeline/sfm.py` (pycolmap, CPU, ~5 min/1000 frames) → `pipeline/mapper.py` (nerfstudio splatfacto, GPU, ~10 min) → `maps/{mission_id}/splat.ply`. **Sequential in v1** — SfM must complete (pose_status=success) before nerfstudio starts; nerfstudio is invoked asynchronously (non-blocking for search). Parallel SfM+Mapper deferred to v2.

**DINOv3 availability:** `dino` Qdrant named vector stored only when `MODEL_NAME=dinov3`. "Find more like this" and change detection primary path use `dino`; fallback to `clip` when unavailable. Active learning formula requires DINOv3; if `MODEL_NAME=openclip`, `active_learning_score` uses CLIP embedding distance only.

**Coordinate frame:** ENU (East-North-Up), ROS REP-103. Quaternion: `(qx,qy,qz,qw)` unit quaternion, body→ENU. Global ENU origin fixed at creation time from first GPS-valid mission. Valid ~50km radius; multi-site support v2.

## Implementation Details

### 1. Mission Summary Report
- **Module:** `pipeline/report_generator.py`
- **Trigger:** end of `VideoIndexer.run()` after active tagging completes
- **Output:** `reports/{mission_id}/summary.html` (Jinja2 template; no PDF — YAGNI)
- **Content:** mission metadata, top-5 frames by `active_learning_score` (thumbnails + captions), 3D camera path screenshot (plotly PNG inline), uncertainty histogram, `al_tag` distribution
- **Storage:** `missions.report_path` updated

### 2. Multi-Mission Change Detection
- **Module:** `pipeline/change_detection.py`
- **Trigger:** post-pipeline step after `pose_status=success`
- **Algorithm:** Qdrant GPS bounding box filter (payload range index on `gps.lat`, `gps.lon`) → find prior-mission frames in area → compute CLIP or DINOv3 cosine distance → flag pairs above `CHANGE_DETECTION_THRESHOLD` (env var; default: `0.35` for CLIP, `0.25` for DINOv3)
- **Qdrant payload indexes:** `CREATE PAYLOAD INDEX ON gps.lat (float range)` and `gps.lon (float range)` at collection init. Scale: designed for ≤100 missions × 500 keyframes = 50K points; Qdrant handles this efficiently.
- **Latency note:** validate 2D range filter performance (lat AND lon simultaneously) with synthetic 50K-point collection. Fallback: pre-filter lat only, post-filter lon in Python.
- **Output:**
  ```sql
  CREATE TABLE change_detections (
    id SERIAL PRIMARY KEY,
    mission_id_a TEXT REFERENCES missions(mission_id),
    mission_id_b TEXT REFERENCES missions(mission_id),
    frame_id_a TEXT REFERENCES frames(frame_id),
    frame_id_b TEXT REFERENCES frames(frame_id),
    pose_distance FLOAT4,
    embedding_distance FLOAT4,
    change_score FLOAT4,
    detected_at TIMESTAMPTZ DEFAULT now()
  );
  ```
- **API:** `GET /missions/{id}/changes`. Streamlit page: side-by-side frame pair viewer.

### 3. Robot-Facing Pose Query API
- **Module:** `app/routers/robot.py`
- **Endpoint:** `POST /query/pose`
- **Request:**
  ```json
  {"lat": float|null, "lon": float|null, "alt": float|null,
   "tx": float|null, "ty": float|null, "tz": float|null,
   "heading_deg": float, "radius_m": float (default 50), "top_k": int (default 5)}
  ```
- **Priority logic:**
  - GPS present (`lat`+`lon`) → Qdrant GPS bounding box pre-filter, then optional `tx/ty/tz` reranking
  - Only `tx/ty/tz` present (no GPS) → **not supported in v1; return HTTP 400** "GPS coordinates required for v1 spatial query; tx/ty/tz-only path deferred to v2"
  - Neither present → HTTP 400 "lat/lon required"
- **Response:** `{frames: [...], query_ms: int}` (frame objects include frame_id, mission_id, timestamp_ms, caption, active_learning_score, al_tag, pose, gps)
- **Latency target:** p99 < 200ms (advisory use)
- **Auth:** `X-API-Key` header (existing pattern)

### 4. Embedded 3DGS Viewer
- **Module:** `ui/components/splat_viewer.py`
- **Serving:** nginx (new Docker Compose service) exposes `DATA_DIR/maps/` as `/static/maps/`. SuperSplat served as static files on a separate port. URLs configured via env vars `STATIC_SERVER_URL` (default `http://localhost:8080`) and `SUPERSPLAT_SERVER_URL` (default `http://localhost:8090`).
- **Implementation:** `st.components.v1.iframe(f"{SUPERSPLAT_SERVER_URL}/?url={STATIC_SERVER_URL}/static/maps/{mission_id}/splat.ply")`
- **Fallback:** if no `splat.ply` (GPU unavailable / reconstruction pending), show status message and pycolmap sparse point cloud download link

### 5. "Find More Like This"
- **Extend:** `app/services/search.py` + Streamlit results page
- **Implementation:** Qdrant `search()` on `dino` (or `clip` fallback) embedding of clicked frame; exclude source frame; limit 20
- **UI:** `st.button("Find more like this")` per result card; results in expandable section

### 6. Persistent Global Map — Phase 1 (GPS point cloud)
- **When GPS present:** convert lat/lon/alt → ENU metric frame (origin = first GPS-valid mission's mean position, fixed at `global_map` creation). Store `{tx,ty,tz,qx,qy,qz,qw}` in `frames.global_pose_json` and Qdrant payload.
- **When GPS absent:** log warning, skip global map registration, frames remain locally searchable
- **`registration_transform_json`:** 4×4 SE3 rigid body matrix from local mission ENU to global ENU (GPS-derived; deterministic). `registration_error` = NULL for Phase 1.
- **Phase 2 (v2 — ICP fusion):** see TODOS.md
- **DDL:**
  ```sql
  CREATE TABLE global_map (
    global_map_id TEXT PRIMARY KEY,
    origin_lat FLOAT8, origin_lon FLOAT8, origin_alt FLOAT8,
    created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ, mission_count INT DEFAULT 0
  );
  CREATE TABLE global_map_missions (
    id SERIAL PRIMARY KEY,
    global_map_id TEXT REFERENCES global_map(global_map_id),
    mission_id TEXT REFERENCES missions(mission_id),
    registration_transform_json JSONB,
    registration_error FLOAT8,   -- NULL for Phase 1 GPS registration
    registered_at TIMESTAMPTZ
  );
  ```

## Reviewer Concerns

Issues unresolved after 3 adversarial review iterations. Implementer must resolve before shipping.

1. **al_tag precedence rule missing** (Clarity) — If a frame ranks in top-50 by `active_learning_score` AND has embedding distance > 0.5 from nearest centroid, both `needs_annotation` and `novel` apply. The implementation must define precedence. Recommended fix: `needs_annotation` takes priority (`if score_rank <= K: needs_annotation; elif dist > 0.5: novel; else: none`).

2. **asyncpg vs psycopg2 split unspecified** (Clarity) — Both drivers are mentioned but no guidance on which code path uses which. Recommended fix: asyncpg for async worker job queue; psycopg2 for sync FastAPI endpoints. Document connection pool size (CPU cores × 2) and timeout (10s, retry 3×).

3. **GPU memory budget absent** (Feasibility) — Florence-2 (~12GB peak on V100), CLIP/DINOv3, and nerfstudio splatfacto all run on the GPU worker. No memory budget or scheduling strategy specified. Risk of OOM. Implementer must profile and document memory constraints before the Dockerfile.nerfstudio is merged.

4. **k-means (k=20) on DINOv3 dimensions unvalidated** (Feasibility) — DINOv3 embeddings are 1536-dimensional. k=20 may be too coarse or too fine for typical mission sizes. Run k-means on a synthetic 1000-frame mission and validate convergence time and cluster quality before hardcoding `k=20`.

5. **Qdrant 2D range filter performance is a pre-ship blocker** (Feasibility) — The plan notes "validate 2D range filter performance with synthetic 50K-point collection" as a latency note. This must be treated as a hard blocker for change detection: validate before architecture is locked, not after.

6. **Pipeline parallelism undefined** (Feasibility) — SFM (pycolmap, CPU-bound, ~5 min/1000 frames) and Mapper (nerfstudio splatfacto, GPU-bound, ~10 min) overlap in resource usage. Whether they run sequentially or concurrently is unspecified. Define worker thread/process model for the reconstruction pipeline.
