## Key Interactions to Verify

### pipeline/florence_model.py (tests/unit/test_florence_model.py, @pytest.mark.gpu)
- `FlorenceModel().__init__()` loads weights successfully (GPU required)
- `caption_batch([1x1 PIL image])` returns `List[Tuple[str, float]]` with confidence in [0.0, 1.0]
- `caption_batch([])` returns `[]` — length invariant for empty input
- `caption_batch(N images)` returns exactly N tuples — length guarantee
- Per-image exception returns `("", 0.5)` for that index, rest continue
- `output_scores` available: confidence is computed, not defaulting to 0.5
- `output_scores` unavailable: graceful fallback to 0.5, not crash
- OOM during batch=16: fallback to batch=1 succeeds

### pipeline/indexer.py post-loop integration (tests/unit/test_florence_integration.py)
- After `index_video()`, `frame_records` have non-NULL `caption` and `caption_confidence`
- `caption_model` = `"florence-2-large"` in each frame_record
- Frame re-load from disk: `Image.open(frame_path)` called for each kept frame
- `FileNotFoundError` during re-load: that frame gets `caption=""`, `caption_confidence=0.5`, continues
- Qdrant `set_payload` called in 128-frame batches after Florence completes
- Qdrant `set_payload` failure: logged warning, no crash, DB still has captions

### scripts/backfill_captions.py (tests/unit/test_backfill_captions.py)
- Frames with `caption IS NOT NULL` are skipped (resume-safe)
- Frames with missing files: log warning + continue, no crash
- `--skip-qdrant` flag: Qdrant set_payload not called
- Empty DB: script exits cleanly with "0 frames to process" log
- DB update writes `caption`, `caption_confidence`, `caption_model`

### scripts/migrate_postgres.py (tests/unit/test_migration.py or test_job_db_postgres.py)
- After migration runs: `frame_facts_json JSONB` column exists on `frames` table
- After migration runs: `caption_model TEXT` column exists on `frames` table
- Migration is idempotent: running twice does not error

## Edge Cases
- `caption_batch` called with exactly `FLORENCE_BATCH_SIZE` images (boundary condition)
- `caption_batch` called with `FLORENCE_BATCH_SIZE + 1` images (triggers second batch)
- Florence OOM at batch=1 (rare): should propagate as exception, worker crashes (fail fast)
- Backfill script run twice: second run is no-op (all frames already captioned)
- Qdrant `set_payload` partially fails (first batch succeeds, second fails): DB consistent, Qdrant partial

## Critical Paths
- New mission indexed end-to-end: Florence captions appear in `frames.caption`, `qdrant` payload has `caption` key
- `active_learning_score` computed correctly when `caption_confidence=0.5` (fallback case)
- `scripts/profile_gpu_memory.py` run with Florence loaded: confirms ≤ 8GB VRAM for Phase 1
