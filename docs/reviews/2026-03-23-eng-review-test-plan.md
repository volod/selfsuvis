## Affected Pages/Routes
- `POST /query/pose` — robot advisory API: GPS validation, top_k cap, auth, Qdrant GPS filter
- `GET /missions/{id}/changes` — change detection results viewer
- Streamlit mission view — 3DGS viewer iframe, "Find more like this", change detection pairs

## Key Interactions to Verify
- Submit video → mission created in PostgreSQL → pipeline completes → al_tags assigned
- "Find more like this" button returns visually similar frames (DINOv3 similarity)
- 3DGS viewer iframe loads splat.ply from nginx (CORS must work)
- Robot API: GPS query returns frames with p99 < 200ms
- Robot API: null lat/lon returns HTTP 400 (not 500)
- Change detection: two missions overlapping GPS area shows change pairs

## Edge Cases
- Mission with 0 GPS-tagged frames: change detection skipped, global map registration skipped
- Mission with fewer frames than k=20: k-means k reduces to n_frames
- MODEL_NAME=openclip: active_learning_score uses CLIP distance only (no DINOv3)
- top_k=10000 on robot API: clamped to 100
- Florence-2 GPU OOM: falls back to batch=1, then CPU if needed
- nerfstudio container unreachable: map_status=skipped (not job failed)
- migrate_postgres.py run twice: idempotent (IF NOT EXISTS on all tables)
- Duplicate video (same SHA-256): skip re-indexing (existing processed_db behavior)

## Critical Paths
- Full mission pipeline: video upload → frames → SfM → embeddings → Florence-2 → active learning → mission complete
- PostgreSQL migration: stop worker → docker compose up postgres → migrate_postgres.py → restart → verify healthz
- Robot API end-to-end: POST /query/pose with valid GPS → Qdrant GPS filter → frame results < 200ms
- Change detection: two missions with overlapping GPS → change_detections table populated
- 3DGS viewer: splat.ply exists → SuperSplat iframe loads via nginx with CORS headers
